leetcode392.判断子序列

2023-08-02
823- 3m
leetcode算法

题目

给定字符串 s 和 t ，判断 s 是否为 t 的子序列。

字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，"ace"是"abcde"的一个子序列，而"aec"不是）。

进阶：

如果有大量输入的 S，称作 S1, S2, … , Sk 其中 k >= 10亿，你需要依次检查它们是否为 T 的子序列。在这种情况下，你会怎样改变代码？

致谢：

特别感谢 @pbrother 添加此问题并且创建所有测试用例。

示例 1：

输入：s = "abc", t = "ahbgdc"
输出：true

示例 2：

输入：s = "axc", t = "ahbgdc"
输出：false

提示：

  • 0 <= s.length <= 100
  • 0 <= t.length <= 10^4
  • 两个字符串都只由小写字符组成。

解答

  • 暴力解法：
class Solution {
public:
    bool isSubsequence(string s, string t) {
        if (s.size() == 0 && t.size() == 0) {
            return true; // 如果 s 和 t 均为空，则 s 是 t 的子序列，返回 true
        }
        int i = 0; // 定义 s 的指针 i，用于遍历 s 的字符
        int temp = -1; // 定义 temp 变量，用于记录上一次匹配的 t 中的位置
        int j = 0; // 定义 t 的指针 j，用于遍历 t 的字符
        for (i = 0; i < s.size(); i++) { // 遍历 s 中的字符
            for (j = temp + 1; j < t.size(); j++) { // 从上一次匹配的位置之后开始遍历 t 中的字符
                if(s[i] == t[j]) { // 如果当前 s 中的字符与 t 中的字符匹配
                    temp = j; // 更新 temp 为当前匹配的 t 中的位置
                    break; // 跳出当前循环，继续下一个字符的匹配
                } else if (j == t.size() - 1) {
                    return false; // 如果 t 中没有字符与 s 中的字符匹配，返回 false
                }
            }
        }
        if (i == s.size() && j < t.size()) {
            return true; // 如果 s 已经遍历完毕，而 t 还有剩余字符，则 s 是 t 的子序列，返回 true
        }
        return false; // 其他情况下，返回 false
    }
};
  • 更高效的方法

我们可以通过一次遍历字符串 t 来判断字符串 s 是否为 t 的子序列。

思路如下：对于字符串 s 和 t，我们维护两个指针 i 和 j，分别指向 s 和 t 的开头。然后，我们逐个比较 s[i] 和 t[j]，如果相等，则将两个指针都向后移动一位，如果不相等，则只移动 t 的指针 j。最终，如果 s 的指针 i 移动到末尾，说明 s 是 t 的子序列，返回 true，否则返回 false。

class Solution {
public:
    bool isSubsequence(string s, string t) {
        int i = 0;
        int j = 0;
        while (i < s.size() && j < t.size()) {
            if (s[i] == t[j]) {
                i++; // 如果当前字符匹配，移动 s 的指针
            }
            j++; // 不论当前字符是否匹配，移动 t 的指针
        }
        return i == s.size(); // 如果 s 的指针移动到末尾，说明 s 是 t 的子序列，返回 true；否则返回 false
    }
};
